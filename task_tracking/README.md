# GraphNetExtractAgent 任务跟踪系统

> 📋 集中管理所有任务，避免重复工作
>
> **重要原则**: 执行任何任务前必须检查任务列表，执行后必须立即更新状态！

---

## 任务执行流程（铁律）

```
开始任务前 → 检查任务是否已被分配/完成
    ↓
执行任务时 → 移动任务文件到in_progress/并标记状态
    ↓
完成任务后 → 立即移动到completed/或failed/，更新状态
```

## 任务分配规则

1. **开始任务前**
   - 查看 pending/ 目录
   - 确认任务未被他人领取
   - 检查依赖任务是否完成

2. **领取任务时**
   - 将任务文件移动到 in_progress/
   - 更新状态为 `in_progress`
   - 添加执行者信息

3. **完成任务后**
   - 将任务文件移动到 completed/ 或 failed/
   - 更新状态为 `completed` 或 `failed`
   - 记录完成时间和结果

4. **任务失败时**
   - 移动到 failed/
   - 记录失败原因
   - 标记是否可以重试

---

## 注意事项

⚠️ **禁止行为**:
- 不检查任务列表直接开始工作
- 完成任务后不更新状态
- 多人同时执行同一任务

✅ **推荐行为**:
- 定期查看任务列表（每30分钟）
- 及时更新任务状态
- 复杂任务拆分为子任务
